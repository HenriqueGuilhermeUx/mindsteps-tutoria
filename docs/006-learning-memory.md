# Learning Memory Specification

Version 1.0 Draft

## Purpose

Learning Memory is the long-term pedagogical memory of MindSteps.

Unlike chat history, it stores educational knowledge about the learner that improves future teaching decisions.

## Principle

The platform should remember learning, not conversations.

## What Should Be Remembered

- concepts mastered;
- fragile concepts;
- misconceptions;
- successful pedagogical strategies;
- preferred analogies;
- learner interests over time;
- confidence evolution;
- review history;
- family participation;
- teacher interventions.

## What Should NOT Be Remembered

- unnecessary personal conversations;
- sensitive information unrelated to learning;
- permanent labels about intelligence or ability.

## Memory Types

### Semantic Memory
Concepts the learner understands.

### Procedural Memory
How the learner solves problems.

### Strategy Memory
Which teaching approaches worked best.

### Interest Memory
Themes that increase curiosity and engagement.

### Reflection Memory
Insights produced by learner self-reflection.

## Memory Lifecycle

Learning Event
→ Evaluation
→ Memory Update
→ Cognitive Twin Update
→ Future Recommendations

## Design Rules

1. Memory supports learning, not surveillance.
2. Every stored element must have educational value.
3. Memory is continuously revised.
4. Old assumptions can be replaced by new evidence.
5. Learners continue to evolve.

## Product Goal

Every new learning session should start from accumulated understanding rather than from zero.
